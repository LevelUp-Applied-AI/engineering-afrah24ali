# PR Self-Review Checklist

Before opening any PR, make sure the following are true:

- [ ] Code correctness — the code does exactly what the PR title says
- [ ] Test coverage — new code has tests and all existing tests pass
- [ ] Documentation — README or other docs updated to reflect changes
- [ ] Scope — this PR contains only one logical change; no mixing features, refactors, or docs
- [ ] No debug artifacts — no temporary print statements or breakpoint() calls left
- [ ] Dependencies — any new libraries are necessary and documented in README or requirements.txt
- [ ] Style & formatting — code is clean, well-formatted, and follows project conventions
